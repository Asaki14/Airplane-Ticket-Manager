## Layout Contract

- Breakpoints: `0-767px` mobile, `768-1023px` tablet, `1024px+` desktop.
- Public shell uses a mobile-first single-column layout up to 767px, then expands to a centered content frame with maximum content width of 1200px.
- Public top area must feel travel-oriented: atmospheric hero band or scenic color field first, utility navigation second. Do not render the homepage shell like an analytics dashboard.
- Admin `/admin` reuses the same tokens but removes decorative background treatments; surfaces stay flatter and denser than the public shell.
- Admin list view is stacked cards on mobile and a table on desktop. Do not use a dense data table below 1024px.
- CMS form is single-column on mobile and two-column grouped sections from 1024px upward. Group fields into: fare basics, travel timing, rule summary, source and freshness, recommendation.
- Primary actions in admin stay in a sticky footer/action bar on mobile and a sticky top-right action cluster on desktop.

---

## Interaction Contract

- Public and admin live in one app, but `/admin` must read as a restricted internal surface rather than a separate product. Navigation labels and typography remain consistent across both.
- Deal lifecycle states are fixed as `draft`, `published`, and `expired`. Use neutral styling for `draft`, accent styling for `published`, and muted low-emphasis styling for `expired`.
- Publishing is manual. `发布` stays disabled until all required structured fields and freshness timestamps are valid.
- Expired deals must disappear from public-facing lists automatically once expiry time passes; in admin they remain visible with an `expired` state chip and the expiry timestamp.
- Whenever a deal summary is shown publicly in Phase 1, freshness metadata order is fixed as `发布时间 -> 更新时间 -> 失效时间`.
- Form validation appears inline at field level, with one summary message at the top after submit failure. Do not rely on toast-only validation.
- Unsaved edits require a leave-confirmation dialog: `你有未保存的更改，确定离开吗？`
- Focus treatment is mandatory on all keyboard-reachable controls: 2px accent ring with 2px offset.
