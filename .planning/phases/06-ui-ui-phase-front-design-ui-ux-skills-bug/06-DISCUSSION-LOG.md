## 卡片统一体系

### Q1: 首页 DealCard 高度策略

| Option | Description | Selected |
|--------|-------------|----------|
| 统一最小高+摘要截断 | 统一 min-height，长文案按行数截断，列表视觉更整齐且利于体系统一 | ✓ |
| 完全自适应高度 | 不截断内容，但卡片高度差会更明显 | |
| 分级高度（推荐/普通） | 仅重点卡放大，层次更强但维护更复杂 | |

### Q2: 跨页面卡片结构规范

| Option | Description | Selected |
|--------|-------------|----------|
| 统一骨架：头-主体-底部 | 列表卡、对比卡、详情信息卡共用三段式结构 | ✓ |
| 各页面独立设计 | 页面独立优化但一致性风险更高 | |
| 只统一外观不统一结构 | 低成本，但一致性提升有限 | |

**User's choice:** 统一最小高+摘要截断；统一骨架：头-主体-底部。
**Notes:** 用户核心诉求是先解决卡片大小不一致，避免页面间风格割裂。

## 详情页布局修复

### Q1: 详情页主布局方案

| Option | Description | Selected |
|--------|-------------|----------|
| 桌面 2 列 + 内容分组重排 | 移动端单列，桌面双列，直接缓解高窄问题 | ✓ |
| 保持 3 列仅加宽容器 | 改动小，但高窄问题可能残留 | |
| 桌面单列长文流 | 阅读连贯但对比效率下降 | |

### Q2: 票规翻译模块呈现

| Option | Description | Selected |
|--------|-------------|----------|
| 拆成多个短卡 + 关键信息优先 | 行李/退改/经停等分卡展示，长文本摘要或折叠 | ✓ |
| 保留单大卡完整展示 | 一次看全，但容易又高又窄 | |
| tab 切换内容 | 更省空间，但降低信息可见性 | |

**User's choice:** 桌面 2 列 + 内容分组重排；票规翻译短卡化。
**Notes:** 用户明确指出详情页当前卡片“高度很高、长度很窄”，本区决策以直接修复该问题为优先。

## 春日视觉语言

### Q1: 春天气氛强度

| Option | Description | Selected |
|--------|-------------|----------|
| 轻春日（清新克制） | 低饱和春色 + 柔和背景层次，优先可读性 | ✓ |
| 中等春日（明显氛围） | 主题更明显，需更强对比控制 | |
| 强主题春日（海报感） | 视觉冲击大，但信息界面风险高 | |

### Q2: 春日元素落点（多选）

| Option | Description | Selected |
|--------|-------------|----------|
| 页面背景与 Hero 区 | 在页面大层次建立季节氛围 | ✓ |
| 卡片边框与标签 | 在卡片细节强化统一风格 | ✓ |
| 按钮与交互态 | 在交互状态体现主题语气 | ✓ |
| 插画/装饰图形 | 轻量装饰提升氛围表达 | ✓ |

**User's choice:** 轻春日强度；四类区域全部覆盖。
**Notes:** 方向是“有春天旅游气氛但不过度喧宾夺主”。

## 信息密度与交互节奏

### Q1: 列表卡信息密度

| Option | Description | Selected |
|--------|-------------|----------|
| 决策优先：核心字段+次要折叠 | 保持决策效率，避免信息拥挤 | ✓ |
| 高密度全量展示 | 信息全但可能造成拥挤和高度波动 | |
| 极简展示 | 清爽但可能增加跳转成本 | |

### Q2: 动效边界（初选）

| Option | Description | Selected |
|--------|-------------|----------|
| 仅关键反馈动效 | 轻量稳健，强调反馈 | |
| 丰富动效 | 视觉更活泼，性能和维护风险更高 | ✓ |
| 几乎无动效 | 风险低但提升感知有限 | |

### Q3: 丰富动效的范围细化

| Option | Description | Selected |
|--------|-------------|----------|
| 中高强度 | 页面入场/分区渐入/卡片按钮反馈，避免重视差和长时动画 | ✓ |
| 高强度 | 增加滚动联动和更强装饰动画 | |
| 丰富但可降级 | 默认丰富并强制弱设备降级 | |

**User's choice:** 决策优先密度；动效采用中高强度丰富方案。
**Notes:** 在美化目标下提升动感，但仍要求布局和阅读效率优先。
